Vue 3 Composition API 实战与响应式原理

发布于 2026-02-24

Vue 3 的 Composition API 提供了一种更灵活的代码组织方式，尤其适合复杂组件。同时，Vue 3 基于 Proxy 的响应式系统比 Vue 2 的 Object.defineProperty 更强大。

1. Composition API 核心用法

<script setup>
import { ref, reactive, computed, onMounted } from 'vue'

const count = ref(0)
const state = reactive({ name: 'Vue', version: 3 })
const double = computed(() => count.value * 2)

function increment() {
  count.value++
}

onMounted(() => {
  console.log('组件已挂载')
})
</script>

2. 响应式原理：Proxy 与 Reflect

Vue 3 使用 new Proxy(target, handler) 代理整个对象，可以拦截属性的读取、设置、删除等操作。配合 Reflect 完成默认行为，从而精确追踪依赖。

const reactive = (obj) => {
  return new Proxy(obj, {
    get(target, key, receiver) {
      track(target, key)  // 依赖收集
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      const result = Reflect.set(target, key, value, receiver)
      trigger(target, key) // 触发更新
      return result
    }
  })
}

3. 实战：可复用的组合式函数

将逻辑抽离为 useXxx 函数，可以在多个组件间共享状态和行为。

// useCounter.js
import { ref } from 'vue'
export function useCounter(initial = 0) {
  const count = ref(initial)
  const inc = () => count.value++
  const dec = () => count.value--
  return { count, inc, dec }
}

掌握 Composition API 能极大提升代码的可维护性和复用性，是现代化 Vue 开发的必备技能。
