Webpack 5 性能优化：代码分割与懒加载

发布于 2026-04-03

Webpack 5 提供了强大的代码分割能力，可以将第三方库、公共模块和按需加载的页面拆分成独立的 chunk，从而减少首屏加载时间。

1. 入口分割（Entry Points）

配置多入口时，每个入口会生成独立的 bundle。

module.exports = {
  entry: {
    main: './src/main.js',
    admin: './src/admin.js'
  }
}

2. 使用 SplitChunksPlugin 提取公共代码

optimization: {
  splitChunks: {
    chunks: 'all',
    cacheGroups: {
      vendor: {
        test: /[\\/]node_modules[\\/]/,
        name: 'vendors',
        priority: 10
      },
      common: {
        minChunks: 2,
        name: 'common',
        priority: 5
      }
    }
  }
}

3. 动态导入（懒加载）

使用 import() 语法实现路由级别的懒加载。

// 路由配置
const routes = [
  {
    path: '/about',
    component: () => import('./views/About.vue')
  }
]

4. 预加载与预获取

Webpack 支持魔法注释：/* webpackPrefetch: true */ 和 /* webpackPreload: true */ 来优化资源加载时机。 

import(/* webpackPrefetch: true */ './Modal.vue')

5. 其他优化技巧

  • 使用 terser-webpack-plugin 压缩 JS。
  • 使用 css-minimizer-webpack-plugin 压缩 CSS。
  • 设置 devtool: 'source-map' 仅在需要时启用。
  • 使用 cache: { type: 'filesystem' } 持久化缓存。

通过合理配置 Webpack，可以让你的应用加载速度提升数倍，用户体验大幅改善。
