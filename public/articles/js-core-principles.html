JS 的一些常用底层原理

发布于 2025-07-15

JavaScript 的底层原理并不只是面试题，它们会直接影响我们写业务代码时的判断与设计。真正理解原型链、作用域链、闭包、this 和事件循环，能让很多“玄学 bug”变得非常可解释。

1. 原型链

JavaScript 的对象查找属性时，会先查当前对象自身，如果没有，再沿着原型链逐级向上查找，直到 null 为止。这也是很多继承实现方式的基础。

function Person(name) {
  this.name = name
}
Person.prototype.sayHi = function () {
  console.log('Hi, ' + this.name)
}

2. 作用域链与闭包

函数在定义时就确定了自己的词法作用域。闭包的本质，就是函数可以记住并访问它定义时所在作用域中的变量。

function createCounter() {
  let count = 0
  return function () {
    count++
    return count
  }
}

3. this 指向

this 的指向取决于函数调用方式，而不是定义位置。常见绑定规则有默认绑定、隐式绑定、显式绑定和 new 绑定。

4. 事件循环

JavaScript 是单线程语言，异步任务依赖事件循环调度。宏任务和微任务的执行顺序，是我们处理 Promise、定时器和 DOM 更新时必须掌握的基础。

console.log(1)
setTimeout(() => console.log(2))
Promise.resolve().then(() => console.log(3))
console.log(4)

输出顺序是：1、4、3、2。

5. 为什么这些原理很重要

当项目复杂度上来之后，理解底层原理能帮助我们写出更稳定、更易维护的代码，也能帮助我们在性能优化和问题排查时更快地定位根因。
