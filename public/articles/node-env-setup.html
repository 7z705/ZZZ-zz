Windows/Mac 下配置 Node.js 开发环境的完整指南

发布于 2025-11-22

无论是做 Vue、React 还是 Node 服务端开发，Node.js 环境都是前端工程化的基础。一个稳定的环境，可以让后续项目开发顺畅很多。

1. 安装 Node.js

Windows 推荐下载安装官方 LTS 版本；Mac 可以使用官网安装包，也可以使用 Homebrew 安装。

brew install node

2. 验证是否安装成功

node -v
npm -v

3. 推荐配置镜像

国内开发环境建议为 npm 配置稳定镜像，以提升依赖安装速度。

npm config set registry https://registry.npmmirror.com

4. 版本管理建议

如果你需要维护多个项目，建议使用版本管理工具来切换不同 Node 版本。这样可以避免旧项目与新项目之间的环境冲突。

5. 常见问题

常见问题包括环境变量未生效、权限不足、镜像源失效、项目依赖锁文件冲突等。排查时优先确认 Node 与 npm 版本、镜像配置以及项目目录中的锁文件。

6. 最终建议

尽量统一团队 Node 版本，使用 LTS 版本开发，并在项目中通过文档说明推荐版本，这样能减少环境问题带来的沟通成本。
